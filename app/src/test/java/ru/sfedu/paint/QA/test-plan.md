# План тестирования мобильного приложения Paint

## 1. Объект тестирования
Мобильное приложение для рисования (Android). Основные функции: создание холста через шаблоны/пользовательский размер, рисование (карандаш/ластик/линейка), палитра цветов с заменой цвета, управление толщиной и прозрачностью, undo/redo, очистка холста, сохранение в локальную галерею, экспорт PNG/JPEG, поиск/переименование/удаление рисунков в галерее.

## 2. Объем тестирования
Входит:
- Создание холста: пресеты и пользовательский размер.
- Рисование: карандаш, ластик, линейка.
- Палитра и замена цвета (двойной тап по цвету).
- Управление толщиной и прозрачностью.
- История действий: undo/redo.
- Очистка холста.
- Сохранение рисунков (локальная галерея приложения).
- Экспорт PNG/JPEG.
- Экран галереи: поиск, открытие, переименование, удаление.
- Корректность работы с ориентацией экрана и жизненным циклом (auto‑save при сворачивании).
- Нефункциональные проверки: производительность, стабильность, совместимость, UX‑риски.
- Юнит‑тесты в `app/src/test/java/ru/sfedu/paint` (ConvertersTest, HistoryManagerTest, JsonCompressionTest, PathSerializerTest).

## 3. Типы тестирования
- Функциональное: холст, инструменты, палитра, диалоги, галерея, экспорт.
- Регрессионное: после изменений в логике рисования/хранилища.
- Smoke: открыть галерею → создать холст → нарисовать → сохранить.
- UI/UX: корректность жестов (tap/double‑tap/drag), видимость иконок.
- Тестирование производительности: отклик при рисовании, время экспорта.
- Совместимость: разные версии Android, размеры экранов, ориентации.
- Надежность/стабильность: длительная сессия, частые undo/redo, переход в фон.
- Модульное: запуск юнит‑тестов из `app/src/test/java/ru/sfedu/paint`.

## 4. Среда тестирования
Устройства:
- Эмуляторы: Android 10, 12, 14 (минимум 2 размера экрана).
- Физические устройства: 1 смартфон.
- ПО: Последняя сборка из `main`.
- Данные: чистая установка и сценарии с существующими рисунками.

## 5. Критерии входа
- Сборка успешно проходит `assembleDebug`.
- Функциональные требования зафиксированы.
- Доступны тестовые устройства/эмуляторы.
- Нет критических дефектов в текущей версии.

## 6. Критерии выхода
- Нет блокирующих/критических дефектов.
- Высокие дефекты устранены или согласованы.
- Пройдены smoke и основные функциональные сценарии.
- Документированы найденные дефекты и результаты тестов.

## 7. Риски и допущения
Риски:
- Регрессии в логике истории (undo/redo) и очистки холста.
- Потеря рисунков при сбоях/смене ориентации или выходе в фон.
- Снижение производительности на слабых устройствах при длинных штрихах.
- Ошибки экспорта (формат PNG/JPEG, ошибка записи файла).
- Некорректная работа жестов (double‑tap на названии/палитре).

Допущения:
- Приложение не использует сеть.
- Рисунки и метаданные хранятся локально.
- Экспорт реализован через системные API и может возвращать ошибку без падения.
